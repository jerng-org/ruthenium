    - `docker pull amazon/dynamodb-local`

        --name dynamodb-local \
        --network lambda-local \
        -p 8000:8000 \
        amazon/dynamodb-local -jar DynamoDBLocal.jar -sharedDb
        `

            on the same network

          specifying named bridge network for (sam local) to connect to later,
          and using the `-sharedDb` parameter in order to avoid the need to
          specify (access keys) for invocations
